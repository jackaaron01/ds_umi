## 背景

昨日完成了键盘遥操作 Tab 录制功能、显示和抖动修复。但机械臂在**不按键时仍会飘移**——根本原因是 IK 求解器对相同目标位姿产生不同的解（多解振荡）。

### 已尝试的修复

| 修复 | 效果 |
|------|------|
| 增大 MuJoCo 阻尼 (2→10) | 部分改善 |
| 降低伺服增益 (kp 500→200) | 部分改善 |
| 增大低通滤波 (alpha 0.3→0.7) | 缓解但未根除 |
| 仅按键时发布位姿 | 完全不动（过度） |

### 根因分析

```
keyboard_teleop (30Hz) → 相同 wrist_pose
  ↓
hand_mapper: IK求解(T_target, q_init=q_current)
  ↓  q_current = 上一帧的解
  ↓  IK 有多个解 → 不同帧选不同解 → 振荡
safety → MuJoCo
```

**核心问题**：hand_mapper 对相同输入持续重算 IK，IK 用上一帧的 `q_current` 做种子，产生不同的解，形成自激振荡。

### 1. 修复 IK 振荡（优先级：高，预计 1 小时）

**方案 A**：在 hand_mapper 中跳过无变化的 IK 求解
- 缓存上一个目标位姿 `T_last`
- 如果 `pose_error(T_current, T_last) < threshold`，复用上一个关节解
- 改动位置：`stage_1/teleop_bridge/hand_mapper.py`

**方案 B**：在 keyboard_teleop 中发送增量
- 仅在按键时更新目标位姿，松手后停止发布
- 让 hand_mapper 保持最后一个有效解

**方案 C**：固定 IK 初始种子
- 不用 `q_current`（上一帧解），改用固定种子
- 缺点：可能收敛到不同的分支

**推荐方案 A**：最小改动，直接解决问题。
