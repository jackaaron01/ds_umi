## 明日任务

### 1. Docker 镜像更新（优先级：高，预计 30 分钟）

`docker/Dockerfile` 已添加 `mujoco` 包，但尚未 `make rebuild`。旧镜像每次 `make shell cmd="..."` 仍需 `sudo pip install -q mujoco`。

- [ ] `make rebuild` 构建新镜像
- [ ] 验证 `python3 -c "import mujoco; print(mujoco.__version__)"` 无需手动安装
- [ ] 验证仿真管道测试 (`test_sim_pipeline.py`) 在新镜像中开箱即用

### 2. MJCF 模型精度修复（优先级：高，预计 2-4 小时）

当前 `xarm6.xml` 是简化通用 6-DOF 臂，与 xArm6 物理几何不符。FK 输出与手写 IK 不匹配，仿真数据不能直接用于训练。

- [ ] 选项 A：从 xArm6 URDF 生成精确 MJCF（需解决 xacro → URDF 转换的 ROS2 包路径问题）
- [ ] 选项 B：用 DH 参数重算 body 相对位姿（修正 `generate_mjcf.py` 中 `compute_relative` 的坐标系错误）
- [ ] 验证：MuJoCo FK 与手写 FK 误差 <1mm（当前 >1m）
- [ ] 验收：仿真环境中 `solve_ik(T_target)` → `move_joints(q_sol)` → `get_joint_state()` 的 FK 闭环自洽

### 3. Recorder 显示 bug 修复（优先级：中，预计 15 分钟）

`_finalize_episode` 中 `Episode 0 saved (0 steps)` 显示有误：
- 实际有 200+ 步写入，但 `step_count` 显示 0
- 根因可能是 `_finalize_episode` 中 `self._writer` 已被 replace 或 `end_episode()` 清零了 `step_count`

- [ ] 修复显示逻辑，让保存日志输出正确的 step count
- [ ] 验证录制后输出的步数与 HDF5 文件实际步数一致

### 4. Stage 3 ACT 基线准备（优先级：中，预计 1-2 小时）

全规划 Stage 3 第一项："ACT 是最成熟的 demonstration learning 基线"。

- [ ] 确认 ACT 训练脚本对数据格式的要求（chunk size、obs/action 维度）
- [ ] 用 v3.0 转换器产出一个 mock 数据集（100+ episodes），验证可被 ACT 加载
- [ ] 确认训练环境依赖（PyTorch 版本、CUDA 可用性）已在 Docker 镜像中就绪
- [ ] 如果快速：跑通一个最小 ACT 训练（mock 数据 + 1 epoch），验证训练回路

### 5. 仿真可视化（优先级：低，如果做完以上）

| 优先级 | 任务 | 理由 |

| 1 | 镜像更新 | 消除每次手动安装 mujoco 的重复劳动 |
